열기 옵션 6가지
ios::binary
ios::app:
    뒤에 새로운 내용을 붙여 쓰게 됩니다. 
    원본 파일은 건들이지 못함
ios::ate:
    자동으로 파일 끝에서 부터 읽기와 쓰기를 실시합니다.
    즉 파일을 열 때 위치 지정자가 파일 끝을 가리키고 있게 됩니다
    따라서 위치 지정자를 움직이면 원본 파일이 변경될 수 있다.
ios::trunc:
    파일 스트림을 열면 기존에 있던 내용들이 모두 지워집니다.
    기본적으로 ofstream 객체를 생성할 때 이와 같은 설정으로 만들어집니다.
ios::in, ios::out:
    ifstream, ofstream 객체를 생성할 때 각각은 이미 설정되어 있음.

std::ofstream의 기본값 = Truncate (내용 비우기) std::ofstream은 파일을 열 때,
std::ios::app (추가) 또는 std::ios::in (읽기) 플래그가 명시적으로 지정되지 않으면
기본적으로 std::ios::trunc 플래그가 활성화됩니다.
즉, 파일을 여는 즉시 파일의 모든 내용이 지워집니다.
